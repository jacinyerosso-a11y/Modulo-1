actualizamos los repositorios y instalamos apache

sudo apt update
sudo apt install apache2


primera parte

Creamos archivos de virtualhosting para el puerto 80 (Hola Mundo)

cd /etc/apache2
cd sites-available
nano holamundo.conf 

agregamos nuestro host virtual

<VirtualHost *:80>
    ServerAdmin admin@midominio.local
    ServerName holamundo.local
    ServerAlias www.holamundo.local
    DocumentRoot /var/www/holamundo.local

    <Directory /var/www/holamundo.local>
        Options Indexes FollowSymLinks
        AllowOverride All
        Require all granted
    </Directory>

    ErrorLog ${APACHE_LOG_DIR}/holamundo_error.log
    CustomLog ${APACHE_LOG_DIR}/holamundo_access.log combined
</VirtualHost>


actualizamos 

a2ensite holamundo.conf
systemctl reload apache2


Agregamos contenido al virtualhosting web

nano /var/www/holamundo.local/index.html

<html>
<title>
Hola Mundo
</title>
<body>
<h1>
Hola Mundo
</h1>
</body>
</html>

Incluimos hola.local en el /etc/hosts

nano /etc/hosts

Agregamos

127.0.0.1 holamundo.local

Probamos con curl o con nuestro navegador

######################Practica 2############################


actualizamos y instalamos

sudo apt update
sudo apt install postfix mailutils libsasl2-modules -y

vamos a la carpeta de configuracion


sudo nano /etc/postfix/main.cf

modificamos los ajustes con estos scrip 

# CONFIGURACION BASICA
myhostname = localhost
mydomain = localhost
myorigin = $mydomain

# REDES Y DESTINOS
inet_interfaces = localhost
mynetworks = 127.0.0.0/8 [::1]/128
mydestination = $myhostname, localhost

# RELAY GMAIL
relayhost = [smtp.gmail.com]:587

# AUTENTICACIÓN GMAIL
smtp_sasl_auth_enable = yes
smtp_sasl_password_maps = hash:/etc/postfix/sasl_passwd
smtp_sasl_security_options = noanonymous
smtp_sasl_mechanism_filter = plain

# SEGURIDAD TLS
smtp_use_tls = yes
smtp_tls_security_level = encrypt
smtp_tls_CAfile = /etc/ssl/certs/ca-certificates.crt

# ALIASES
alias_maps = hash:/etc/aliases
alias_database = hash:/etc/aliases


luego borramos la linea smtp_tls_security_level=may

luego vamos a esta ruta

sudo nano /etc/postfix/sasl_passwd
